# Case-study template

Use this structure for any new project:

1. Honest status and one-sentence system summary
2. Problem and target user
3. Solution and personal contribution
4. Architecture as concrete system boundaries
5. State workflow from input to evidence-bearing result
6. Technical decisions and the hardest challenge
7. Test/build evidence with exact counts where verified
8. Security and safety boundaries
9. Video or live evidence only when publicly verifiable
10. Limitations, roadmap, related work, and next system

Do not infer users, business outcomes, production scale, employment, team size, ownership, or deployment status from code alone.
