# Security policy

Report a suspected vulnerability privately through GitHub's security-advisory feature for this repository. Do not include secrets or personal data in a public issue.

The site is a static portfolio: it has no authentication, database, forms, analytics, cookies, or server-side application API. Its main security boundaries are dependency integrity, external media, browser policy headers, content accuracy, and deployment configuration. See `docs/SECURITY_REVIEW.md` for the reviewed controls and residual risks.
